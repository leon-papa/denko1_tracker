# プルダウン配色とレーダーチャート表示の修正報告 (2026/05/08)

## 1. 課題内容
一部の環境（特定のブラウザやOSの組み合わせ）において、以下の表示上の不具合が発生しました。
1. **プルダウンメニューの視認性**: 「新規記録をつける」画面で、試験期や判定を選択するプルダウンの文字が背景と同化（白背景に白文字）し、選択肢が見えなくなる。
2. **レイアウトのズレ**: ダッシュボードのレーダーチャートが中央に配置されない。

## 2. 原因分析
1. **ブラウザデフォルトスタイルの干渉**: `option` 要素の背景色が指定されていなかったため、一部のブラウザがシステムデフォルトの背景色を採用し、アプリ側で指定した白文字と同化してしまった。
2. **センタリング設定の不足**: `margin: 0 auto` だけでは、特定の画面幅やグリッドレイアウト内において、ブラウザの解釈により中央寄せが不完全になる場合があった。

## 3. 実施した修正
`src/main/resources/static/css/style.css` を修正しました。

### プルダウン配色の固定
- `option` 要素に対し、アプリのテーマカラーに合わせたダークな背景色（`#1e293b`）と文字色（`white`）を明示的に指定。
- ブラウザやOSの設定に左右されず、常に高いコントラストを確保。

### レーダーチャートの中央寄せ強化
- `chart-container` に対し、`display: flex`, `justify-content: center`, `align-items: center` を追加。
- flexbox を用いることで、内部のキャンバス要素を確実にコンテナ中央へ配置。

## 4. 修正結果
- ユーザー環境（ブラウザ、スマホ等）を問わず、プルダウンメニューの文字が正しく表示されることを確認。
- ダッシュボードのレーダーチャートが期待通り中央に配置されることを確認。
