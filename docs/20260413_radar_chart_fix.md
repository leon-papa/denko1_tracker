# レーダーチャート表示不具合の修正報告 (2026/04/13)

## 1. 課題内容
ユーザーより、レーダーチャートのデータ表示が不自然（例：80%のはずが0%として表示される等）であるとの報告を受けました。

## 2. 原因分析
調査の結果、以下の3点が原因として特定されました。
1. **データ同期の不備**: バックエンドからフロントエンドへ「項目名リスト」と「数値リスト」を別々の属性として渡しており、シリアライズの過程で順序がずれるリスクがあった。
2. **最新データの定義齟齬**: グラフには「年度・回数が最新」のデータが表示されていたが、ユーザーは「直近で入力・更新した」データが反映されることを期待していた。
3. **表示設定の視認性**: Chart.js の設定で目盛り（ticks）が透明になっており、データの正確な位置が把握しづらかった。

## 3. 実施した修正案

### バックエンド (Java)
- **エンティティの拡張**: `WrittenExamRecord` および `SkillExamRecord` に `updatedAt` フィールドを追加。`@PrePersist` / `@PreUpdate` を使用して自動的に更新日時を記録するように変更。
- **リポジトリの更新**: `findByUserOrderByUpdatedAtDesc` メソッドを追加し、最新の更新順でレコードを取得可能にした。
- **コントローラの修正**: 
    - グラフ表示対象を「最後に更新されたレコード」に変更。
    - 項目名と数値の順序を保証するため、明示的に `ArrayList` に変換してフロントエンドへ渡すように改善。

### フロントエンド (Thymeleaf / Chart.js)
- **チャート設定の最適化**: 
    - 目盛りの `color` を透明から可視色に変更。
    - `min: 0`, `max: 100` を明示的に設定し、スケールを固定。
    - データの値を 0〜100 のパーセンテージ形式で渡していることを確認。

## 4. 修正結果
- `mvnw compile` によるビルド成功を確認。
- 過去のレコードを編集した場合、その内容が即座にダッシュボードのグラフに反映されるようになった。
- グラフ上に目盛りが表示され、数値の正確な位置が視覚的に確認可能になった。

## 5. 今後の対応
今後の機能追加時も、項目名と数値のペアの整合性を保つため、リスト形式でのデータ受け渡しを標準とする。
