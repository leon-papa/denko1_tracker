# 全体実行計画：Denko1-Tracker

講師の実践的手法（フェーズを区切って着実に進めるスタイル）に基づき、以下の実行計画（ロードマップ）を策定します。
1つのフェーズが完了して動作検証をパスするごとに、次のフェーズへと移行します。

## Phase 1: 基礎環境構築とデータベース設計 [完了]
「データを保存する箱と、Spring Bootの土台を作る」工程です。
* `denko1-tracker` プロジェクトの自動生成。
* H2 Database の設定。
* ドメイン層（エンティティ）の作成。
  * `User` (ユーザー)
  * `WrittenExamRecord` (筆記試験レコード)
  * `SkillExamRecord` (技能試験レコード)
* データベースへの保存・取得を行う `Repository` の作成。

## Phase 2: テスト駆動によるビジネスロジック開発 [完了]
「記録を計算して、弱点を判定したりグラフ用のデータを作る」工程です。
* `Service` クラスの作成。
* **弱点分析ロジック**: 各カテゴリの入力得点から「正答率（%）」を割り出し、一番低い項目を特定。
* JUnitを用いた自動テストコードの作成（堅牢性の担保）。

## Phase 3: 画面の作成（フロントエンド統合） [完了]
「ユーザーが実際に見る画面を作る」工程です。
* Thymeleafを用いたHTMLテンプレート群の作成。
* JavaScript（Chart.js）を組み込み、レーダーチャートとして可視化。
* モダンスタイル（Glassmorphism）の適用。

## Phase 4: セキュリティと機能の総仕上げ [完了]
「ユーザーごとにデータを見分けるための鍵をかける」工程です。
* Spring Securityの導入と、ログインシステムの実装。
* ログイン中のユーザーのデータだけを集計するように全体を修正。

## Phase 5: デプロイと運用性拡張 [完了]
「実際の試験勉強で使いやすく、運用しやすくする」工程です。
* **Render (PostgreSQL) への移行**: 
  * `EnvironmentPostProcessor` による JDBC URL の動的生成。
  * 本番環境用プロファイル（`application-prod.properties`）の整備。
* **データの可搬性 (CSV Export)**: 
  * Excel等での文字化けを防ぐための BOM (Byte Order Mark) 付与。
  * `Written` / `Skill` それぞれの全記録エクスポート機能。
* **モバイル最適化 (UX Fix)**: 
  * `Viewport` メタタグの全画面適用。
  * メディアクエリによるスマホ用レスポンシブ UI（余白・レイアウト）の構築。
  * スマホログイン時の 403 エラー（プロキシ経由の CSRF 対策）の解消。

## Phase 6: 完全なCRUD対応とアカウント管理 [NEW / 進行中]
「ポートフォリオとしての完成度と実用性を極める」工程です。
* **削除機能の実装**: 
  * 筆記試験レコードおよび技能試験レコードの個別削除。
  * `POST` メソッドによる安全な削除処理と所有権チェック（他人のデータは消せない）。
  * 削除時の「確認ダイアログ」の実装（誤操作防止）。
* **編集機能の実装**: 
  * 既存レコードの修正。
  * `record.html` の再利用による編集フォームの構築。
* **ユーザープロパティ管理**:
  * ユーザー表示名の変更機能。
  * アカウントの退会（全データ削除）機能。
* **UI/UXのリファイン**:
  * スマホ版ヘッダーのボタン配置の最適化（ログアウトボタンの押しやすさ向上）。

## 検証プラン (Phase 6)
1. **個別削除検証**: レコードが削除され、ダッシュボードのグラフが即座に再計算されるかを確認。
2. **不正操作ガード**: 別ユーザーのレコードIDを直接リクエストして削除を試み、拒否されることを確認。
3. **編集検証**: 既存の点数を書き換え、変更が永続化されることを確認。
