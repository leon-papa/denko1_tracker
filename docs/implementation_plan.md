# 全体実行計画：Denko1-Tracker

講師の実践的手法（フェーズを区切って着実に進めるスタイル）に基づき、以下の実行計画（ロードマップ）を策定します。
1つのフェーズが完了して動作検証をパスするごとに、次のフェーズへと移行します。

## Phase 1: 基礎環境構築とデータベース設計
「データを保存する箱と、Spring Bootの土台を作る」工程です。
* `denko1-tracker` プロジェクトの自動生成。
* H2 Database の設定。
* ドメイン層（エンティティ）の作成。
  * `User` (ユーザー)
  * `WrittenExamRecord` (筆記試験レコード：年度、カテゴリ別得点)
  * `SkillExamRecord` (技能試験レコード：公表問題番号1〜10、タイム、合否判定)
* データベースへの保存・取得を行う `Repository` の作成。

## Phase 2: テスト駆動によるビジネスロジック開発
「記録を計算して、弱点を判定したりグラフ用のデータを作る」工程です。
* `Service` クラスの作成。
* **弱点分析ロジック**: 各カテゴリの入力得点から「正答率（%）」を割り出し、一番低い項目を特定するプログラムを実装。
* 👉 **AI活用ポイント**: この計算ロジックが絶対に間違っていないことを証明するために、**「JUnitを用いた自動テストコード」** を作成し、システムの堅牢性を担保します。

## Phase 3: 画面の作成（フロントエンド統合）
「ユーザーが実際に見る画面を作る」工程です。
* Thymeleafを用いたHTMLテンプレート群の作成。
* JavaScript（Chart.js等）を組み込み、Phase 2で計算した正答率データを「レーダーチャート」として画面上に美しく表示する。
* 技能試験のタイム履歴テーブルの作成。

## Phase 4: セキュリティと機能の総仕上げ
「ユーザーごとにデータを見分けるための鍵をかける」工程です。
* Spring Securityの導入と、簡易的なログインシステムの実装。
* ログイン中のユーザーのデータだけを集計するように全体を修正。
* 実際のデータを使って一通りの結合テスト（手動）を行う。
