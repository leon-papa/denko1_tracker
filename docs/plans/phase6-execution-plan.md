# Phase 6 実行計画：完全なCRUD対応とアカウント管理 [進行中]

本フェーズでは、システムをポートフォリオとして「完成」に導くため、データの個別管理（編集・削除）とユーザープロファイル機能の実装を行い、実用性と堅牢性を極めます。

## 1. データの個別削除機能 (Delete)
*   **バックエンド**: 
    *   `POST /written/delete/{id}`
    *   `POST /skill/delete/{id}`
    *   セッション情報を用いた所有権チェックの実装（他人のデータ削除を防止）。
*   **フロントエンド**:
    *   ダッシュボードのテーブルに削除ボタンを配置。
    *   JavaScript (`window.confirm`) による確認ダイアログの実装。

## 2. データの個別編集機能 (Update)
*   **設計方針**: 現在の `record.html` を「編集モード」に対応させるため、コントローラーから `Model` を渡して初期値を埋め込む。
*   **バックエンド**:
    *   `GET /record/edit/{id}` : IDに基づきデータを取得し、入力画面を表示。
    *   `POST /record/update` : 修正された内容をデータベースへ反映。
*   **UX**: 編集ボタンをクリックした際、どのレコードを修正しようとしているのか明示されるように設計。

## 3. ユーザープロファイル管理
*   **表示名変更**: ログイン後の「ようこそ、〇〇さん！」の名称をユーザーが自由に変更できる機能。
*   **パスワード更新**: セキュリティ向上のためのパスワード変更画面の作成。
*   **退会機能（アカウント削除）**: 
    *   ユーザー情報および関連する全ての学習記録を一括で安全に削除する機能。
    *   Spring Data JPA の `CascadeType.ALL` または手動による一括削除ロジックの構築。

## 4. UI/UX の最終ブラッシュアップ
*   **モバイルヘッダーのリファイン**: 横幅の狭いデバイスでも「新規記録」と「ログアウト」ボタンが押しやすく、バランス良く配置されるよう維持・調整。
*   **フィードバックの充実**: 保存完了時や削除完了時に、トースト通知（メッセージ表示）が出るように改善を検討。
