# [Phase 4] セキュリティとマルチユーザー対応

Phase 3 でアプリのメイン機能が動くようになりました。
最終フェーズとなる Phase 4 では、一時的に無効化していた**Spring Securityを再有効化**し、複数のユーザーがそれぞれのデータを安全に管理できる状態を構築してMVPを完成させます。

## 1. 構成内容

### Spring Security の再有効化と設定
一時的に除外していた設定を解除し、カスタムの認可設定およびログイン・ログアウト設定を行います。
* **権限設定**: `/register` および静的リソースは全員許可。その他（トップ画面や記録）はログイン必須。
* **ログイン画面**: 全体のGlassmorphismテーマに合わせたカスタム画面を使用。

### ユーザー認証システムの実装
データベース（`User` エンティティ）を参照して認証を行うプロバイダーを作成します。
* `CustomUserDetailsService`: ユーザー検索と権限付与。
* パスワードのハッシュ化（BCrypt）。

### データのユーザー紐付け (マルチユーザー対応)
これまでのダミーID（1L）を廃止し、ログイン中のユーザー情報をバックエンドの全処理（保存・取得・計算）で利用するように修正します。

## 2. 実装ファイル詳細
1. `Denko1TrackerApplication.java`: 除外設定の削除
2. `SecurityConfig.java`: 認可フィルタチェーンの設定
3. `CustomUserDetailsService.java`: 認証ロジック
4. `AuthController.java`: ログイン・登録制御
5. `WebController.java`: `Principal` を用いた個人データ処理への変更
6. `login.html`, `register.html`: 新規画面

## 3. 動作確認項目
* ログインなしで `/` にアクセスした際、ログイン画面へリダイレクトされること。
* 新規ユーザーを登録し、そのユーザーでログインできること。
* 別のユーザーでログインしたとき、他人の記録が表示されないこと。
